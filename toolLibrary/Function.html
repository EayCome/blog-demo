# 1. 函数相关

# 1.1 call()&apply()&bind()

# 1.1.1 API说明

  • call()
    • 语法: call(fn,obj,...args)
    • 功能: 执行fn,使this指向obj,并将后面n个参数传入给fn,方法等同于Function构造函数上的call方法
  • apply()
    • 语法: call(fn,obj,args)
    • 功能: 执行fn,使this执行obj,并将args数组中的元素传给fn,方法等同于Function构造函数上的bind方法
  • bind()
    • 语法: bind(fn,obj,...args)
    • 功能: 给fn绑定this为obj,并指定参数为后面的n个参数,方法等同于Function构造函数上的bind方法

# 1.1.2 实现说明

  • 区别call()/apply()/bind()
    • call(obj)/apply(obj): 调用函数,指定函数中的 this 为第一个参数的值
    • bind(obj): 返回一个新的函数,新函数内部会调用原来的函数,且 this 为 bind 指定的第一个值
    • 注意: 如果 obj 是 null/undefined, this 指向 window

# 1.1.3 编码实现

  • 自定义实现函数对象的call方法: src/function/call.js
    function call(fn,obj,...args){
        // 判断 obj 是否是 undefined 或者 null
        if(obj === undefinedd || obj === null){
            obj = window
        }
        // 创建一个临时函数
        obj.tmp = fn;
        // 通过 obj 来调用这个方法 执行fn函数 this自然指向 obj
        const result = obj.tmp(...args);
        // 删除obj上的临时方法
        delete obj.tmp;
        // 返回fn执行的结果
        return result;
    }
  • 自定义实现函数对象的apply方法: src/function/apply.js
    function apply(fn,obj,args) {
        // 判断 obj 是否是 undefined 或者 null
        if(obj === undefinedd || obj === null){
            obj = window
        }
        // 创建一个临时函数
        obj.tmp = fn;
        // 通过 obj 来调用这个方法 执行fn函数
        const result = obj.tmp(...args);
        // 删除obj上的临时方法
        delete obj.tmp;
        // 返回fn执行的结果
        return result;
    }
  • 自定义实现函数对象的bind方法: src/function/bind.js
    function bind(fn,obj,...args){
        obj.tmp = fn;
        return (...args2)=>{
            return call(fn,obj...args,...args2)
        }
    }

# 1.2 函数节流与函数防抖

# 1.2.1 相关理解

  • 事件频繁触发可能造成的问题?

    • 一些浏览器事件: window.onresize、window.onmousemove等,触发的频率非常高,多次调用会造成页面卡顿
    • 多次向服务端发送请求,频繁触发,服务器压力增大

  • 如何限制事件的频繁触发?

    • 函数节流
    • 函数防抖

  • 函数节流(throttle)

    • 理解:
      • 在函数需要频繁触发时: 函数执行一次后，只有大于设定的执行周期后才会执行第二次
      • 适合多次事件按时间做平均分配触发
    • 场景:
      • 窗口滚动(scroll)
      • 窗口调整(resize)
      • DOM 元素的拖拽功能实现
      • 抢购疯狂点击

  • 函数防抖(debounce)

    • 理解
      • 在函数需要频繁触发时,在规定时间内,只让最后一次执行,前面的不失效
    • 场景
      • 浏览器引擎搜索联动(keyup/input)

# 1.2.2 API说明

  • throttle() 节流
    • 语法: throttle(callback,wait)
    • 功能: 创建一个节流函数,在 wait 毫秒后最多执行callback 一次
  • debounce() 防抖
    • 语法: debounce(callback,wait)
    • 功能: 创建一个防抖动函数,该函数会从上一次被调用后, 延迟 wait 毫秒调用 callback

# 1.2.3 编码实现

  • src/function/throttle.js: 函数节流
    function throttle(callback,wait){
        let start = 0;
        return function(event){
            // 只有当上次处理的时间超过了wait时,才会执行处理事件
            let current = Date.now()
            if(current - wait > wait) {
                callback.call(this,event); // 需要制定this和参数
                start = current;
            }
        }
    }
  • src/function/debounce.js: 函数防抖
    function debounce(callback,wait) {
        // 定义一个计数器
        let timer = -1;
        return function(event) {
            if(timer !== -1){
                clearTimeout(timer);
            }
            // 启动延迟 await 时间后执行的定时器任务
            timer = setTimeout(function() {
                callback.call(this,event)
                // 启动完后重置标识
                timer = -1
            },wait)
        }
    }
